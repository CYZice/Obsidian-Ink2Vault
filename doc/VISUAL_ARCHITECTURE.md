# PDF 处理方案 - 可视化对比

## 当前问题的可视化分析

### 现状：为什么 PDF 无法转换

```
🔴 问题链：

用户选择 PDF
    ↓
代码读取 PDF 整个文件到内存
    ↓ (100MB PDF → 100MB 内存使用)
转换为 Base64
    ↓ (100MB → 133MB 字符串，内存膨胀 33%)
发送给 OpenAI Vision API
    ↓ API 尝试解析 Base64 为图像
    ↓ (❌ 失败：PDF 不是有效的图像格式)
    ↓
❌ 错误：500 API Error
    └─ "decode base64 image data failed: missing RIFF chunk header"

结果：
✗ 转换失败
✗ 内存占用 230MB+ （可能导致 OOM）
✗ 用户看不到进度
✗ 无法恢复（整个中止）
```

### 改进后（支持 PDF）

```python
def convertFile(filePath):
    if filePath.endswith('.pdf'):
        ✅ 支持（新增）
        return convertPdfStream(filePath)
    elif filePath.endswith(('.jpg', '.png')):
        ✅ 支持（保留）
        return convertSingleImage(filePath)

def convertPdfStream(filePath):
    buffer = readFile(filePath)
    pdf = pdfjsLib.getDocument(buffer)
    totalPages = pdf.numPages
    
    markdownParts = []
    
    for pageNum in range(1, totalPages + 1):
        try:
            # 1. 渲染单页
            page = pdf.getPage(pageNum)
            canvas = renderPage(page, scale=1.5)
            showProgress(pageNum, totalPages)
            
            # 2. 转图片
            jpeg = canvas.toDataURL('image/jpeg', 0.8)
            
            # 3. 发送给 AI（不等待全部）
            result = ai.convert(jpeg)
            
            # 4. 追加到 Markdown
            markdownParts.append(f"## 第 {pageNum} 页\n\n{result}")
            
            # 5. 清理内存（关键！）
            page.cleanup()
            canvas.clear()
            
        except Exception as e:
            markdownParts.append(f"> [!ERROR] 第 {pageNum} 页失败: {e}")
            continue  # 继续处理下一页
    
    # 6. 合并所有结果
    finalMarkdown = "\n\n---\n\n".join(markdownParts)
    saveFile(filePath, finalMarkdown)
    return finalMarkdown
```
